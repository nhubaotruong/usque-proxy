<objective>
Rewrite the forwarding loops and reconnect logic in usque-bind/bind.go to match usque-android's proven pattern.

Purpose: The current forwarding loops send ALL errors to errChan (causing unnecessary reconnects on single packet failures), and the reconnect loop uses exponential backoff up to 60s (causing long outages). This plan ports usque-android's error classification (CloseError = fatal, others = log+continue) and constant 1-second reconnect delay.

Output: Modified `usque-bind/bind.go` with new `forwardUp`, `forwardDown`, and `maintainTunnel` logic.
</objective>
